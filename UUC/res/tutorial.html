UUC pro vás provede rozměrovou analýzu a pokud vstup nesouhlasí s cílovými jednotkami, dostanete varování. Výsledek výpočtu pak ignorujte.

To je často způsobeno záměnou symbolu jednotky – např. C a F je coulomb a farad, nikoliv stupeň Celsia a Fahrenheita, ty jsou °C a °F: příklad A, příklad B

Také to může být způsobeno syntaktickou chybou – např. mK znamená milikelvin, nikoliv metr krát kelvin, ten by byl zapsán jako m*K nebo m K.

Pozn.: takto zapsaná veličina s rozměrem teploty je vždy zpracována jako rozdíl teploty, nikoliv jako teplota. Jak převést teploty mezi sebou se dozvíte později.

UUC will perform dimensional analysis for you, and if the input doesn't match the target units, you will get a warning. In that case ignore the conversion result.

That's often caused by a misunderstood unit symbol – e.g. C and F are coulomb and farad, not degree Celsius and Fahrenheit, those are °C and °F: example A, B

It can also be caused by a syntax error – e.g. mK means milikelvin, not metre times kelvin, which would be written as m*K or m K.

Note: a quantity with dimension of temperature written this way will always be processed as temperature difference, not as temperature. You will learn later how to convert temperatures.

Jak již bylo zmíněno, program chápe teplotu jako teplotní rozdíl, nikoliv absolutní teplotu (příklad). Program by nemohl poznat, zda-li myslíte T či ΔT, proto se obecně pracuje s ΔT. Speciální zápis pomocí {složených závorek} umožňuje interpretovat teplotu jako absolutní, např. takto: °F na K či °F na °C

Další příklady: °API hustota na SI tam a zpět,
výpočet hustoty vzduchu pomocí K, či právě s využitím {°C}

Pomocí {} lze též použít speciální funkci – přirozený logaritmus, lze využít např. pro logΔT. Samozřejmě, ln není jednotka a proto nemůže být použit mimo {}

Pozn. {složené závorky} na české klávesnici: pravý Alt + B, N

Pozn. jsou zde omezení: ve Vstupu může v {} být jedno číslo a jedna jednotka, v Cílových jednotkách je použití ještě omezenější – celé pole musí být tvořeno pouze tímto výrazem, a nesmí v něm být číslo. Nelze použít prefix ani mocninu. Místo čísla ovšem může být bezrozměrný výraz v závorce, viz logΔT výše.

As mentioned before, the program understands temperature as temp difference, not as absolute temp (example). The program couldn't tell if you want T or ΔT, that's why it generally operates with ΔT. Special syntax with {curly brackets} allows to interpret temperature as absolute, like this: °F to K or °F to °C

More examples: °API density to SI there and back,
air density calculation using K, or using {°C}

Using the {} you can also use a special function – the natural logarithm, for example logΔT. Of course, ln is not a unit and cannot be used outside {}

Note: {curly brackets} on english keyboard: Shift + { } next to Enter

Note: there are limitations: in Input there can only be one number, one unit in {}, in Target units it is even more strict – whole field must comprise only this expression, and there mustn't be a number. Neither prefix nor power can be used. But instead of number you can use dimensionless (bracket), see logΔT above.
